# Contributing Guidelines

Thank you for your interest in contributing!

## How to Contribute

1. **Fork the repository** - Click the fork button.
2. **Clone your fork** - `git clone https://github.com/YOUR_USERNAME/issues-fest.git`
3. **Create a branch** - `git checkout -b fix/issue-name`
4. **Make changes** - Fix one issue at a time.
5. **Commit** - Use clear commit messages.
6. **Push** - `git push origin fix/issue-name`
7. **Pull Request** - Submit a PR referencing the issue.

## Commit Message Guidelines

- Use present tense ("Add feature" not "Added feature").
- Start with a capital letter.
- Keep the first line under 50 characters.
- Reference issue numbers.

## Code Standards

- Write clean, readable code.
- Add comments where necessary.
- Follow existing code style.
- Test your changes.

## Reporting Bugs

If you find a bug not already listed in the Issues tab, please create a new issue!

## Development Workflow

1. Create a feature branch from `develop`.
2. Make your changes.
3. Submit a pull request to `develop`.
4. After review, merge into `develop`.
